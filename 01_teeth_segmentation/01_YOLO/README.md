# Сегментация зубов на ОПТГ с использованием архитектуры YOLO

**Цели исследования**: 
- разработка подхода к сегментации зубов с присвоением номеров по системе FDI на основе архитектуры YOLO;
- достичь следующих значений метрик на отложенной тестовой выборке: mAP50 ≥0.95, Dice ≥ 0.85; 
- обеспечить устойчивую сегментацию зубов на ОПТГ;

Архитектура YOLO (You Only Look Once) хорошо подходит для сегментации зубов на ортопантомограммах с присвоением каждому зубу уникального номера по международной системе FDI (32 метки класса) благодаря ряду ключевых особенностей:
- Обнаружение и классификация объектов.    
YOLO изначально разработана для задачи детекции объектов: она одновременно предсказывает координаты ограничивающих рамок (bounding boxes) и вероятности классов для всех объектов на изображении за один проход нейросети (одноэтапный детектор). Это соответствует сценарию, где на снимке присутствует множество зубов, каждый из которых должен быть не только локализован, но и отнесен к одному из 32 классов (номеров). 
- Возможность сегментации экземпляров (instance segmentation).    
Современные версии YOLO расширены для задачи сегментации экземпляров. Они предсказывают не только рамки, но и попиксельные маски для каждого обнаруженного объекта. Это позволяет выделить контур каждого зуба, отделив его от соседей и окружающих структур (фона в нашем случае), и одновременно присвоить маске соответствующий номер класса. 
- Высокая скорость и эффективность.    
YOLO обеспечивает быстрое обучение и получение результатов, быстрый инференс на новых изображениях. Это достигается за счет сквозного обучения и отсутствия отдельных этапов (как в двухэтапных детекторах типа Mask R-CNN).    
- Способность обрабатывать большое количество объектов на одном изображении.     
На ОПТГ присутствуют зубы верхней и нижней челюсти (до 32 объектов). YOLO эффективно работает со множеством объектов благодаря механизму предсказания на сетке (grid) и постобработке (Non-Maximum Suppression). YOLO сразу предсказывает финальные рамки на основе сетки, что позволяет данной архитектуре работать быстрее, по сравнению с конкурентами.
- Аугментации и устойчивость к вариациям.    
Архитектура YOLO хорошо сочетается с аугментациями данных (повороты, масштабирование, изменения яркости и т.д.), что важно для улучшения качества обучения.

## Методика (дизайн эксперимента)

**Варьируемые условия**:
- архитектуры YOLO (v8, v11, v26);
- размер весов моделей: Nano, Medium, Large, XLarge;
- наличие и тип аугментаций: без аугментаций / HSV-искажения + геометрические трансформации, подробнее об аугментациях для моделей на базе архитектуры YOLO [здесь](https://github.com/drSever/MIPT_X-RayDent/tree/master/01_teeth_segmentation/00_Dataset);
- степень предобработки изображений: полный кадр / обрезка краёв на 5% / обрезка на 10–15%;
- используемый датасет: teeth-seg-3537 Computer Vision Model (автор Godento2) / teeth Computer Vision Dataset (автор Chosun University), подробнее об этих датасетах [здесь](https://github.com/drSever/MIPT_X-RayDent/tree/master/01_teeth_segmentation/00_Dataset);
- настройки оптимизации: по умолчанию / кастомные параметры AdamW;

**Измеряемые показатели**:
- функции потерь: box_loss, cls_loss, dfl_loss, seg_loss
- основные метрики: mAP50, mAP50-95
- дополнительные метрики: Dice, IoU в различных агрегациях (micro, macro, global, по изображениям);
- визуальная оценка на новых неразмеченных данных;

**Фиксированные условия**:
- вычислительная среда: Google Colab (GPU T4, L4, A100) с фиксированным seed;
- количество эпох для основных сравнений: 100 (финальная модель обучалась на 1000 эпох);

**Критерии достижения целей**:
- достижение значений метрик на отложенной тестовой выборке: mAP50≥0.95, Global Dice ≥ 0.85;
- модель демонстрирует приемлемое качество на новых, ранее не размеченных снимках (отсутствие критического переобучения, оцениваемое экспертно).

**Последовательность экспериментальных шагов**:
- подготовка данных: датасеты уже разделены на обучающую, валидационную и тестовую подвыборки;
- обучение baseline-модели (YOLOv8n  с настройками по умолчанию, без аугментаций);
- серия экспериментов с вариацией одной независимой переменной (варьируемое условие) при фиксации остальных;
- сбор метрик и функций потерь после каждого эксперимента;
- сравнительный анализ и выбор оптимальной конфигурации;
- длительное обучение (1000 эпох) лучшей конфигурации;
- финальная валидация на тестовой выборке и новых снимках;

## Методы исследования

**Инструменты**:
- программная среда: Google Colab, Python, фреймворк Ultralytics YOLO.
- аппаратное обеспечение: GPU NVIDIA T4, L4, A100.
- датасеты:
  - основной: teeth-seg-3537 Computer Vision Model (автор Godento2)
  - дополнительный: teeth Computer Vision Dataset (автор Chosun University)

**Параметры оптимизации**:    
Были использованы настройки параметров оптимизации (подобраны эмпирически и скорректированы на основе экспериментов, в основном решено использовать  параметры по умолчанию):
- *оптимизатор*: AdamW - это модификация классического оптимизатора Adam, который сочетает преимущества адаптивных методов с корректной регуляризацией;
- *начальная скорость обучения*: lr0 = 0.001, обычно используется для Adam-подобных оптимизаторов и часто используется при дообучении предобученных моделей;
- *конечный множитель скорости обучения*:  lrf = 0.01, использовано значение по умолчанию, означает, что финальная скорость обучения будет в 100 раз меньше начальной, т.е. 0.00001;
- *momentum* = 0.937, используется в YOLO для оптимизатора AdamW по умолчанию (такое же значение, как и для SGD);
- *weight_decay* = 0.0005, коэффициент L2-регуляризации, который штрафует большие веса, помогая бороться с переобучением (значение по умолчанию);
*warmup_epochs* = 3, техника, при которой в начале обучения скорость обучения постепенно увеличивается от очень маленького значения до lr0, что помогает избежать неустойчивости на первых итерациях (значение по умолчанию);
- *warmup_momentum* = 0.8, обычно в начале обучения используют меньшее значение momentum, чтобы модель быстрее адаптировалась, а затем увеличивают до основного значения (значение по умолчанию);
- *warmup_bias_lr* = 0.1, параметры смещения (bias) во время warmup (значение по умолчанию);

**Анализируемые функции потерь**:
- *box_loss*: показывает, насколько точно предсказанная ограничивающая рамка (bounding box) совпадает с реальным объектом;
- *cls_loss*: оценивает, насколько правильно модель определяет класс объекта внутри каждой рамки (в нашем случае метку класса зуба по FDI);
- *dfl_loss* (Distribution Focal Loss): отвечает за точность границ предсказанной рамки;
- *seg_loss*: насколько точно предсказанная попиксельная маска зуба совпадает с его истинным контуром;

**Метрики оценки**:    
Качество моделей сравнивалось на отложенной тестовой выборке (модель не видела ее при обучении). Использовались метрики для масок Precision, Recall, mAP50, mAP50-95 (встроенная оценка YOLO).     
Дополнительно рассчитывались метрики Dice и IoU:
- *micro*: рассчитывались по всем пикселям всех классов вместе 
- *macro*: среднее арифметическое индивидуальных метрик по классам (если некоторые зубы постоянно хуже сегментируются, то мы это увидим)
- *глобальные*: для всего тестового набора
- *по отдельным изображениям*.   

Дополнительно оценивалось качество обученной модели на новых неразмеченных снимках.

### Итоги

- При оценке loss-функций отмечается склонность моделей к переобучению (seg_loss и dfl_loss) даже на финальной модели. При этом метрики на валидации стабильно растут.
- Сравнение архитектур моделей (на базе размера Medium). При анализе моделей среднего размера в одинаковых условиях обучения (100 эпох) наблюдаются следующие показатели:
  - YOLOv11 M показала лучшие результаты среди новых архитектур с Global DICE 0.8512 / 0.8452 и высоким показателем mAP50 - 0.986, mAP50-95 - 0.733;
  - YOLOv8 M (Baseline) продемонстрировала Global DICE 0.8531 / 0.8466;
  - YOLOv26 M уступила предшественникам с результатом Global DICE 0.8424 / 0.8359, несмотря на использование специфического семантического лосса (sem_loss);
- Сравнение моделей YOLOv8: Nano, M, L, X. Увеличение количества параметров модели дает прирост метрик:
  - Nano: Самая быстрая модель, после кастомной оптимизации достигла Global DICE 0.8442 / 0.8380;
  - M vs L vs X: Переход от Medium (DICE 0.8590) к Large (DICE 0.8602) и Extra Large (DICE 0.8603) дает крайне малый прирост (менее 0.2%), при этом время обучения увеличивается в два раза — с 9.5 до 19.2 часов;
  - при этом mAP50 у всех моделей практически одинаковый ≈0.99, mAP50-95 чуть выше у X-модели ≈0.795; 
- Влияние оптимизации и аугментации:
  - Custom Optimization (AdamW): использование собственных настроек оптимизации стабильно повышает точность. На модели Nano это дало прирост Global DICE с 0.8350 до 0.8442, а метрика mAP50-95 увеличилась с 0.563 до 0.611.
  - Аугментации: применение цветовых искажений (HSV) и геометрических трансформаций (повороты, масштаб) улучшило показатели YOLOv8 M до Global DICE 0.8565 / 0.8503.
- Выделение зоны интереса (кроп). Эксперименты с обрезкой краев снимков (crop) показали, что излишнее удаление контекста вредит сегментации:
  - сильный кроп (10-15%) снизил точность до Global DICE 0.8324;
  - умеренный кроп (5%) показал результат лучше (0.8493), но все же не превзошел работу с полным кадром при разрешении 640 пикселей;
  - использование разрешения 640x640 (разрешение исходных изображений) является оптимальным стандартом для текущего набора данных;
  - даже при частичной обрезке подбородка и височнонижнечелюстных суставов метрики резко падают, хотя при этом не происходит обрезка самих зубов;
- Разрыв между micro/macro и Global/Image показателями DICE/IoU во всех экспериментах незначительны. Это свидетельствует о высокой сбалансированности модели: она одинаково хорошо сегментирует как часто встречающиеся зубы, так и более редкие или сложные классы в системе FDI.
- При оценке модели на новых ортопантомограммах результаты были хуже: 
даже при полном наборе зубов были ОПТГ с неполной разметкой зубов,
еще хуже обстояли дела на изображениях с артефактами (импланты, коронки); края сегментировались грубо, даже если учитывать не сильно точную разметку исходного датасета (см.ниже);

## Выводы
- Использование кастомных настроек AdamW и комбинированных аугментаций даёт больший прирост, чем увеличение размера модели.
- Кроп изображения для выделения зоны интереса малоэффективен и даже может ухудшить метрики, судя по всему глобальный контекст не менее важен, чем локальный для правильного обучения. 
- Использование более точно размеченного, но менее объемного датасета не дает роста качества сегментации зубов.
- Финальный результат: максимально достигнутые метрики зафиксированы у YOLOv11 X на 1000 эпохах — mAP50 0.987, Global DICE (micro/macro) 0.8592 / 0.8529, Global IoU (micro/macro) 0.7532 / 0.7466, что говорит о высоком качестве обнаружении и классификации каждого зуба на изображениях тестовой подвыборки, хотя качество сегментации при этом нельзя назвать сильно высоким.
- Не смотря на неплохие метрики, полученные при анализе модели на тестовой подвыборке, результаты обученной модели на новых данных плохие - модель явно переобучена на исходном датасете.
- Цель по достижению целевых метрик на тестовой выборке выполнена. Однако цель обеспечения устойчивой сегментации на новых данных не достигнута: выявлено критическое переобучение на исходном датасете, что делает невозможным применение модели на практике.
